# 00 — Repository audit (unable.cz)

Snapshot taken 2026-07-17, before implementing the Able.cz dossier.

## Stack
- **Zola 0.22.1** static site generator; single content page + homepage section.
- **Tailwind 3.4.17** (compiled locally/CI), **Flowbite 2.5.2** + Flowbite
  Typography, **Alpine.js 3.15.12** (always-loaded runtime).
- **Chart.js 4.5.1**, **Cytoscape.js 3.34.0**, **p5.js** — loaded **only** on
  `/dossier/` via the template's `scripts` block.
- Node 24; build orchestrated by npm scripts (`css:build`, `js:build`,
  `data:build`, `build`, `verify`).

## Content & templates
- `content/_index.md` → homepage; `content/dossier.md` → `/dossier/`.
- `templates/base.html` is the **only** place metadata is rendered (title,
  description, canonical, OG, Twitter, JSON-LD). Child templates expose no
  metadata hooks — a deliberate anti-drift invariant, enforced by tests.
- `templates/index.html` carries the hero; the word **UNABLE** is the single
  internal link to the dossier (keyboard-focusable, accessible name contains
  visible text).
- `templates/dossier.html` renders the dossier: provenance panel, prose,
  Chart.js/Cytoscape/p5 figures, and always-present data tables.

## SEO / metadata invariant
- Front matter is the single source; `config.toml [extra]` is the fallback
  layer. `seo_type = "Report"` requires an `[extra.evidence]` table (cutoff,
  reviewed_at, review_interval_days, confidence).
- `scripts/seo-validate.mjs` validates the built `public/`: titles, canonicals,
  OG, JSON-LD safety (`seo::j` macro escapes `</script>`), no hardcoded host,
  exactly one `<h1>`, evidence policy on Report pages, orphan detection.
- Robots/sitemap generated by Zola; 404 is `noindex` with no canonical by design.

## Hosting
- GitHub Pages behind Fastly, custom domain via `static/CNAME`. No server, no
  database, no analytics/cookies. Static-only deploy.

## Governance
- `CLAUDE.md` → `AGENTS.md` → `docs/seo.md`. `AGENTS.md` rule 7: never publish a
  claim the evidence does not support. The "Content about real parties" section
  reserves any third-party dossier subject to an explicit, on-record owner
  decision — obtained for Able.cz on 2026-07-17.

## Implication for this work
The dossier is layered onto the existing pattern without new frameworks: JSON
evidence in `data/dossier/able/`, rendered via `load_data`, enhanced by figures
that degrade to tables. No change to the metadata invariant or the deploy model.
