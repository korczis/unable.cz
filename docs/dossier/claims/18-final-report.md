## What was built — IMPLEMENTED & VALIDATED

- **Static route per claim** — `/dossier/claims/clm-01/` … `/dossier/claims/clm-58/`,
  live AND superseded, generated by `scripts/dossier/claims-gen.mjs` (deterministic,
  idempotent, stamped to the evidence cutoff). Each is a **complete static page**
  (no JS needed): breadcrumb, status badge + explanation, the **full proposition**
  (never truncated, byte-identical to canonical — enforced), sources (linked,
  with independence + primary-doc flags), supersession chain (successors/
  predecessors linked), backlinks, provenance, and a link to its JSON.
- **JSON shard per claim** — `/data/dossier/claims/clm-NN.json`, public and
  sanitized (an internal filesystem path leaking through SRC-12's title —
  `cases/DD-Able-CZ-…/artifacts/` — is stripped from the public projection, §18).
- **Claims index** — `/dossier/claims/` with the full inventory + accounting.
- **Real anchor links** — `claims-link.js` turns every `CLM-\d+` in the dossier
  **narrative** into a real `<a href>` to its canonical route (base-path-safe via
  `get_url`). VALIDATED: 49 anchors created; ordinary click navigates, middle/
  Cmd-click open a new tab, address copies, keyboard works. Only IDs present in
  canonical data are linked. Claim-index and claim-page links are server-rendered.
- **Validators wired into the build** — `data:validate` now runs
  `claims-validate.mjs` (routes/JSON exist, counts reconcile, no dangling
  supersession, no internal-path/PII leak, narrative CLM refs resolve).
