# Epistemic impact (PROMPT-09 §10)

See 05-change-model for the ladder. Values: STRENGTHENED, WEAKENED, CORRECTED, CONTRADICTED, RESOLVED, NO_EPISTEMIC_CHANGE, UNKNOWN.

Worked examples from the real ledger:
- r04→r05 claim status upgrades (mirror → primary ARES): STATUS_CHANGED / STRENGTHENED.
- CON-02 (execution contradiction) resolution recorded via claims: RESOLVED.
- SRC-22 (media source) recovery in r06: OBJECT_ENRICHED on the source; preserved evidence unaffected.
- Source availability changes: NO_EPISTEMIC_CHANGE — availability is provider health, not evidence invalidation (§10 rule, unit-tested).
